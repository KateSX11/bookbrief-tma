## 1. Tailwind v4 CSS 变量 + 透明度修饰符不兼容

**问题**：使用 `bg-[var(--tg-theme-bg-color)]/95`、`text-[var(--tg-theme-text-color)]/80` 这类写法，在 Tailwind v4 中会编译为 `color-mix()` 或 `oklch(from ...)` 等现代 CSS 语法。Telegram WebView（尤其老版本）可能不支持，导致样式完全失效，页面看起来是全黑/全白。

**根因**：Tailwind v4 对内置颜色（如 `blue-500/10`）可以正确分解 RGB 通道加 alpha，但对 CSS 自定义变量（`var(--xxx)`）无法分解，只能使用 `color-mix()` 降级，这个 CSS 函数兼容性较差。

**解决方案**：
- **禁止**对 CSS 变量使用 `/opacity` 修饰符
- 需要半透明背景时，直接使用不带透明度的变量：`bg-[var(--tg-theme-bg-color)]`
- 需要较浅文字时，用 Telegram 提供的 hint 颜色：`text-[var(--tg-theme-hint-color)]`
- 需要浅色底时，用二级背景色：`bg-[var(--tg-theme-secondary-bg-color)]`
- Tailwind 内置颜色（如 `bg-emerald-500/10`、`text-white/80`）可以正常使用 `/opacity`

```
✅ bg-[var(--tg-theme-bg-color)]
✅ text-[var(--tg-theme-hint-color)]
✅ bg-white/10    （内置颜色可以用）
✅ from-blue-500/8 （内置颜色可以用）

❌ bg-[var(--tg-theme-bg-color)]/95
❌ text-[var(--tg-theme-text-color)]/80
❌ bg-[var(--tg-theme-button-color)]/10
```

## 2. Telegram WebView 缓存导致部署后看到旧版本/白屏

**问题**：代码推送到 Railway 并部署成功后，Telegram 打开 TMA 仍显示旧版本或空白页面。通过浏览器直接访问 Railway URL 确认内容正常。

**根因**：Telegram 的 WebView 会缓存之前加载的页面（HTML、JS、CSS），即使服务器已更新，WebView 不一定会重新请求。

**解决方案（服务端）**：
在后端返回 `index.html` 时添加禁止缓存的响应头：

```python
return HTMLResponse(
    content=html,
    headers={
        "Cache-Control": "no-cache, no-store, must-revalidate",
        "Pragma": "no-cache",
        "Expires": "0",
    },
)
```

> 注意：`/assets/` 下的 JS/CSS 文件带有 hash 文件名（如 `index-3a4b5c.js`），天然不会有缓存问题，不需要加 no-cache。只有 `index.html` 需要。

**解决方案（客户端临时）**：
- iOS：Telegram 设置 → Data and Storage → Storage Usage → Clear Other Data
- TMA 内：点右上角 `...` → Reload Page
- 终极方案：完全杀掉 Telegram 进程后重新打开

## 3. React 渲染错误导致白屏无提示

**问题**：如果 React 组件渲染时抛出未捕获异常，整个应用直接白屏，没有任何错误提示，用户和开发者都无法判断原因。

**解决方案**：
1. 在 `main.tsx` 中加 `ErrorBoundary` 包裹整个应用：

```tsx
class ErrorBoundary extends Component<
  { children: ReactNode },
  { error: Error | null }
> {
  state: { error: Error | null } = { error: null };
  static getDerivedStateFromError(error: Error) {
    return { error };
  }
  render() {
    if (this.state.error) {
      return <div style={{ padding: 24, color: "#f55" }}>
        <pre>{this.state.error.message}</pre>
      </div>;
    }
    return this.props.children;
  }
}
```

2. 在 `index.html` 的 `#root` 中放一个纯 HTML 加载提示（React 挂载后会覆盖），方便判断 JS 是否加载成功：

```html
<div id="root">
  <div style="display:flex;align-items:center;justify-content:center;height:100vh;">
    載入中…
  </div>
</div>
```

3. `initTelegramApp()` 用 try-catch 包裹，防止 SDK 初始化失败阻断后续代码。

## 4. Git 分支名不一致导致 Railway 部署不生效

**问题**：本地 Git 默认分支是 `master`，但 GitHub 和 Railway 可能跟踪 `main`，导致 `git push` 后 Railway 不触发部署。

**解决方案**：
- 统一使用 `main` 分支：`git branch -M main`
- 修改 GitHub 默认分支：`gh api repos/{owner}/{repo} -X PATCH -f default_branch=main`
- 推送时同时推两个分支以确保万无一失：

```bash
git push origin main
git push origin main:master --force
```

## 5. SQLite 数据库 Schema 变更后旧数据库不兼容

**问题**：修改 SQLAlchemy Model（增删字段）后运行 seed 脚本报错 `table has no column named xxx`。

**根因**：SQLite 不支持 `ALTER TABLE ADD COLUMN` 的复杂操作，`create_all()` 只创建不存在的表，不会更新已有表的结构。

**解决方案**：
- MVP 阶段直接删掉旧数据库文件重建：`rm backend/data/bookbrief.db`
- 删除前需先停掉占用数据库的进程（uvicorn 等）
- 正式项目应使用 Alembic 做数据库迁移

## 6. PowerShell 不支持 `&&` 链接命令

**问题**：在 PowerShell 中使用 `cd xxx && git status` 会报 `ParserError`。

**解决方案**：
- 使用分号分隔：`cd xxx; git status`
- 或分开执行
- 或使用 PowerShell 7+ 的 `&&` 支持
